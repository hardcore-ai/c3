# Personas — SkillWall Live

## Persona 1: Participante (= Votante)

- **Nombre**: Ana, desarrolladora backend
- **Rol**: Participante registrado en la sesión
- **Contexto**: Asiste a un evento técnico remoto desde LatAm, accede via link/QR desde su celular
- **Motivaciones**: Mostrar sus skills, descubrir qué skills tienen otros, interactuar con la comunidad dando likes
- **Frustraciones**: Procesos de registro largos, apps que no funcionan en móvil, latencia alta
- **Capacidades técnicas**: Alta — entiende conceptos técnicos, cómoda con apps web
- **Dispositivo principal**: Smartphone (mobile-first)
- **Interacciones**: Registro (join), upload foto, selección skills, ver wall, dar likes, ver leaderboard

## Persona 2: Admin de sesión

- **Nombre**: Diego, organizador del evento
- **Rol**: Administrador técnico de la sesión
- **Contexto**: Gestiona la sesión del evento via API (curl/Postman), no tiene UI de admin
- **Motivaciones**: Crear sesión antes del evento, limpiar datos al finalizar, asegurar operación fluida
- **Frustraciones**: Complejidad innecesaria, falta de visibilidad operacional
- **Capacidades técnicas**: Alta — cómodo con CLI, curl, APIs REST
- **Dispositivo principal**: Laptop con terminal
- **Interacciones**: Crear sesión, eliminar sesión (y todos sus datos)
